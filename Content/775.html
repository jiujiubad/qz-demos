### 如果一个 div 里所有子元素都有 `float: left;` 这个属性时，这个 div 会发生什么情况？ 

  • 这个 div 会从文本流中脱离出来，浮在其他元素上方，会把后面的元素遮住
  • 解决：在 div 后面紧跟着使用一个 `clear: both;` 来清除浮动
### CSS 属性 `box-sizing` 里面的 `content-box` 跟 `border-box`  的差异？ 

  • 这应该是与 CSS 的盒子模型相关
  • `border` ：指容器最外面的框架；那 `border-box`：应该是指框架的整个边框
  • `content` ：指容器里的内容；`content-box` ：指介于内容与 padding 之间的那一块吧

### n + 1 query 是什么，怎么解决？ 

N + 1 Query 是指将 Rails 指令转成 SQL 语句时，可能会出现一个指令需要查询 N 次的情况 
如： 
`@products.photos` 
 可以通过 `@products.includes(:photos)` 来解决 

### 使用以下 script 给所有注册者寄信时，当规模变大时，服务器会时常当掉，怎么办？ 

``` 
User.all.each do |user| 
    ...
end 
``` 

使用 `User.all.each` 在数据量小的时候（数千条数据）时倒也没有关系，但当数据量上去几万，几十万甚至更多时，就会出问题。 
因为它会抓取一次性将整个表抓过来，给每一行创建一个对象，然后将整个数组留在内存当中。 
这样当数据量大的时候，很可能会导致内存被用完。这也是服务器当机的原因。 

解决的办法： 

  • 使用 `find_each`
    • `User.find_each do |user|`  : 默认情况下每次取 1000 条数据作为一个独立的 model
    • `User.where(weekly_subscriber: true).find_each do |user|` ：与 其他方法关联起来用，如 `where`
    • `User.find_each(start: 2000, finish: 10000, batch_size: 5000) do |user|` :  `find_each` 的选项
  • 使用 `find_in_batches`
    • `User.find_in_batches do |user|` : 也是一次取 1000 条数据，
    • 与 `find_each` 不同的是，`find_in_batches` 是将取出的数据作为一个保存许多 model 的数据（一条数据一个 model ），而不是作为一个独立的 model
