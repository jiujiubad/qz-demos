思路：
学生其实是通过课程和老师链接，通过捞出学生参加的所有课程，就可以进一步捞出学生学习的所有老师
通理，老师通过捞出教授的所有课程，可以进一步捞出所有学生
故，不用直接开一个relationship 的rb 来处理 老师和学生的多对多关系。

1：N:  老师 has many 课程； 课程  belongs_to 老师
N : N： 学生has_and_belongs_to_many 课程
              课程 has_and_belongs_to_many 学生
具体实做如下：
 1. 产生 model, 建立 course_relationship
$rails g model student name:string
$rails g model teacher name:string
$rails g model course name:string
$rails g model course_relationship student_id:integer course_id:integer

$rake db:migrate

2. 设定 teacher 、 course、student 之间的关系
app/models/teacher.rb
class teacher < ApplicationRecord
      ... 
       has_many :courses
end

app/models/course.rb
class course < ApplicationRecord
belongs_to : teacher
has_many : students, :through => :course_relationship, :souce => :student
        ....
end 

app/models/student.rb
class Student < ApplicationRecord
......
has_many :courses, :through => :course_relationship, :souce => :course
......
end

class course_relationship < ApplicationRecord
belongs_to :student
belongs_to : course
end

1.请列出至少7个git指令，并简单说明它们的作用？
  • git clone xxx.git : 从server端clone 代码仓
  • git init : 初始化代码仓
  • git add . :  将专案里面的所有改动（修改的、 新增的、 删除的）加入暂存区
  • git commit -m"xxx"  提交代码到本地代码仓
  • git reset --hard version_number   强制reset代码到指定版本， 如果 HEAD~1， 则是上一个版本
  • git push origin master   提交代码 到服务器
  • git log : 显示历史提交信息
  • git diff ：对比当前代码和上一次提交版本的不同
  • git status: 显示对比上一次提交，所有的更动信息                                          
  • git pull orgin master: 同步server端代码仓最新版本
  • git checkout -b xxx:  新建branch xxx 
  • git checkout xxx： 切换到branch xxx

  1. 添加 shenqiku代码仓地址：      $ git remote add shenqiku  git://github.com/shenqiku .git

        $ git chechout master 【切回自己的master分支】

    5. 提交到自己的远程代码仓 【在master分支】
        $ git push origin master

2.请列举你所知道的几种HTTP status code，并指出他们的意义
2xx （请求成功）表示成功处理了请求的状态代码。
200   （成功）  服务器已成功处理了请求。 通常，这表示服务器提供了请求的网页。 
201   （已创建）  请求成功并且服务器创建了新的资源。 
202   （已接受）  服务器已接受请求，但尚未处理。 
3xx （请求被重定向）表示要完成请求，需要进一步操作。 通常，这些状态代码用来重定向。
4xx （请求错误）这些状态代码表示请求可能出错，妨碍了服务器的处理。
400   （错误请求） 服务器不理解请求的语法。 
401   （未授权） 请求要求身份验证。 对于需要登录的网页，服务器可能返回此响应。 
403   （禁止） 服务器拒绝请求。
404   （Not found未找到） 服务器找不到请求的网页。
5xx（服务器错误）这些状态代码表示服务器在尝试处理请求时发生内部错误。 这些错误可能是服务器本身的错误，而不是请求出错。
500   （服务器内部错误）  服务器遇到错误，无法完成请求。 
501   （尚未实施） 服务器不具备完成请求的功能。 例如，服务器无法识别请求方法时可能会返回此代码。 
502   （错误网关） 服务器作为网关或代理，从上游服务器收到无效响应。 
503   （服务不可用） 服务器目前无法使用（由于超载或停机维护）。 通常，这只是暂时状态。 
504   （网关超时）  服务器作为网关或代理，但是没有及时从上游服务器收到请求。 
505   （HTTP 版本不受支持） 服务器不支持请求中所用的 HTTP 协议版本。

三、前端部分

1.请问如果一个div里面所有的子元素都有float: left这个属性时，这个div会发生什么情况？要如何解？
 float：left 表示元素向左浮动, 如果一个div里面的所有子元素都使用float：left，div内的元素，从左往右依次排列在左侧。
 第二个元素及以后的所有元素，使用clear：left or clear:both，可以使得所有元素纵向排列在左侧。

4.n + 1 query是什么，可以怎么解决？
 n+1 query: 如果需要查询n个对象，那么必须执行n+1次查询语句。因为需要先查询这个class所有item，然后再根据item的id，查询相关内容。
eg：
def index
  @students = Student.includes(:class).all
end

<% @students.each do |student| %>
<tr>
  <td> <%= student.name %> </td>
  <td> <%= student.class.name %> </td>
</tr>
<% end %>

》》》》
student load(0.1m) select * from "students"
class load (0.2m) select * from "class" where "student_id=4"
class load (0.2m) select * from "class" where "student_id=5"
class load (0.2m) select * from "class" where "student_id=6"
class load (0.2m) select * from "class" where "student_id=7"

解决方法：
  • 加一个includes(:class)，Rails 就会智能的将这么多个 query 合并成一个 query 了。


def index
  @students = Student.includes(:class).all
end
student load(0.1m) select * from "students"
class load (0.2m) select * from "class" where "student_id in (4,5,6,7)"
