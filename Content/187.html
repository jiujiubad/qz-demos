def selling_books(q,k)
t = 0
case k
when  1
         t = 100*k*(1-0)+(q-k)*100        
when  2
         t = 100*k*(1-0.05)+(q-k)*100  
when  3
         t = 100*k(1-0.1)+(q-k)*100
when  4
         t = 100*k(1-0.2)+(q-k)*100
when  5
         t = 100*k(1-0.25)+(q-k)*100
end
return t.to_i
end
describe “Total” do
         it “1kind and 1 Quantity,Total = 100” do
                   result = selling_books(1,1)
                   expect(result).to eq(100)
         end
         it “2kind and 2 Quantity, Total = 190” do
                   result = selling_books(2,2)
                   expect(result).to eq(190)
         end
         it “2 kind and 3 quantity, Total = 290” do
                  result = selling_books(3,2)
                   expect(result).to eq(290)
         end
end

心得体会

  • 五万大课的开学典礼
  • 2017-02-02 11:28
听了笑来老师的课程，基本上对全栈的定义有了一个大体的了解。
全栈就和我们现在的学习模式有异曲同工之处，就是要了解整体。重点是要把这件事做出来，例如做一件事情需要6种技能，恰巧这6种技能我都会。而先不要考虑精通，精通是不断练习的结果。而要达到上面的程度，我们要提高学习效率。最快提高效率的方法就是教，教是最好的学。笑来老师用在新东方教英文写作的例子来诠释教是最好的学。
全栈的学习只是展现思想的一种手段，代码只占20%。学的好不好并不重要，能用上才是最重要的。学习全栈工程师的过程当中，最重要的是学习思考模式。解决问题的方法，说明问题的方法，以及创造出我们想要的东西。最开始不可能作出一个非常好的作品。这时我们要退而求其次，然后用教的方式来学习，还时间让别人弄明白，是让自己进步最好的方式。
