作业一

  • 数据库 SQL 入门
  • 2017-06-29 13:33

作业一：
假设我们有一个 Table 叫做 people，里面有字段 id, first_name, last_name, gender (值可能是 M 或 F), birthday
请写出 SQL 捞出所有资料
select * from people


请写出 SQL 捞出所有男生
select * from people where gender="M"


请写出 SQL 插入一笔资料
INSERT INTO people (id, first_name, last_name, gender) VALUES (007, "ma", "junnan", "F");


请写出 SQL 更新 id 为 3 的资料，将姓名修改成你的名字
UPDATE events SET first_name="ma", last_name="junnan" WHERE id=3


请写出 SQL 删除 id 为 4 的资料
Delete * from people where id = 4


补充：
当查询的数据量比较大时，可以增加索引缩短查询时间。
加索引 CREATE INDEX events_user_id_idx ON events(user_id);
这句话的意思时给这个表的user_id加索引而events_user_id_idx就是该表user_id的索引名称on后面是指具体表中的字段。
给people表作索引
create index people_id on people（id）


当然简单的表结构数据不大时一般不需要索引，索引也会占据表空间影响查询速度。

8个问题

  • Growth Hack 实操
  • 2017-03-15 07:26
1。在购买零食小吃“前”，客户前来询问什么问题？
  • 品质有保证吗，是否过期？
  • 运输过程中是否会被损坏
  • 包邮吗
  • 好吃吗
  • 图片跟实物是否相符，分量是否足
  • 客服咨询找谁

2。在网上购买小食的“当天”客户忘记做什么会让使用者体验搞砸？（最常客诉的点）
  • 客服服务态度差
  • 物流慢
  • 种类多的，无从选择
  • 最讨厌收到的时候包的不好，包装对吃的很重要
  • 商品不符合描述，快递慢
  • 退货没人理
  • 发货不及时

3。他们最常做了什么“正确的事”达到很好的体验？
  • 根据场景定制需要，直接选购，省心。
  • 快递当天到
  • 客服随时在，服务周到
  • 收到的超过预期，比图片上还要好
  • 提升生活品质，避免浪费大量时间在购物网站上的筛选，并有爆款商品可以黏住大家的零食欲。
  • 可以帮我打包发公司的零食，省的我去一个个挑

4。他们最常做了什么“错误的事”结果收到很糟的体验？
  • 买回来，吃的时候发现事物掉渣，弄得沙发都是
  • 吃完东西还要洗手
  • 包装太大，一吃不完，放在那里，下次吃味道就变了。
  • 有的食品是保质期很短，忘记吃了，下次吃，发现坏了或者变味道
  • 收到的时候，包装坏了，都不能吃
  • 快递太慢，收到就不想吃了

5。东西卖出后你如何检验他们“做了正确的事”或“做了错误的事”？
  • 有购买后评价
  • 或者调查问卷
  • 客服投诉系统
  • 分享社交网站的评价
  • 收藏宝贝的理由
  • 可以直接问问身边的人购买使用的体验

6。他们如何联络你修正问题？
  • 客服
  • 评价

7。你怎么做事后补偿的instruction? (有FAQ/说明书/部落格/客服专线)
  • 有FAQ，如假包换。
  • 客服专线
  • 有官方微博供吐槽
  • 专门有人回答吐槽评价，作出及时反馈

8。你希望他们如何事后帮你行销?
  • 转发社交平台
  • 写试吃心得，发奖
  • 介绍给朋友同事
