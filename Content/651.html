高频小套路

  • 元学习课2017版
  • 2017-03-13 02:02
我一直从事金融销售行业，分为线上和线下，我以如何推荐自己的报名优惠码为例来给出一个小套路，要提前说的是直接给出对方利润和好处是无效的，这些好处包括学费上的优惠、生硬摆出编程的重要性和明显地借助老师的影响力，这很容易让人产生被人利用的不良反应，他会觉得你让他做这种事是为了你自己，有时候甚至别人觉得你被洗脑了（在我们不了解老师的前提下）。那么我的方法是：1、找到你想要的客户群体，最好是已经有意向报名但是还有顾虑的童鞋（主要是时间短）；2、真诚的与他们沟通，讲出你是如何选择编程并且获得了哪些具体的进步，告诉他们也可以获得这种进步（我强调的是沟通，不要一股脑的自说自话）；3、当他们有意向报名时，和他们分享你的利润，比如充值卡，他们很可能会主动的为你介绍其他童鞋，来帮助你拿到大奖。虽然过程中需要技巧，但再多的技巧也不如别人认为你行或者你值得信任。所以，我真的不明白那些把优惠码摆在一个平台上，就相信别人会从众多的优惠码中选上他的人，是自信还是抱着侥幸心理？

笔试考题 6

  • 求职指南
  • 2017-09-30 23:58
一、lambda, block and Proc的区别？ 
1、block 和 proc都是代码块，Proc可以复用，block不可以复用 
2、block 和 lambda不是类， Proc是类，所以block 和 lambda是小写，Proc是大写 
3、lambda是匿名函数 
4、lambda会对参数个数验证，Proc不会验证 
5、lambda会执行return，Proc遇到return会中断 

二、ruby里的回调方法（钩子方法）有哪些？ 
1、included 
2、extended 
3、prepended 
4、inherited 
5、method_missing 

三、ruby中有哪些动态调用的最佳实践方法？ 
1、用send动态调用 
2、用method来获取某个方法，用call来调用 
3、用eval来动态调用 

四、简单谈谈method_missing的最佳实践方式？ 
method_missing利用的机制是，当一个对象进行某个方法调用的时候，会到其对应的类的实例方法中进行查找。如果没有找到，则顺着祖先链向上查找，直到找到BasicObject类为止。如果都没有则会最终调用一个BasicObject#method_missing抛出NoMethodError异常。 

五、写一个简单的rack？ 
简单来说，Rack是Ruby应用服务器和Rack应用程序之间的一个接口。 
Require “rack” 

class HelloRack 
   def call(env) 
       [200, {“Content-Type” => “text/plain”}, [“Hello Rack!”] 
   end 
end 

Rack::Handler::WEBrick.run HelloRack.new 

六、alias_method 和 alias 的区别？ 
1、alias是Ruby的一个关键字因袭使用的时候是 alias :new name :oldname 
而 alias_method 是 module 类的一个方法，因此使用的时候是 alias_method :newname, :oldname 有一个逗号 
2、alias的参数可以使用函数名或者符号，不能是字符串 
alias_method 的参数可以是字符串，或者符号。 
3、alias_method 可以重定义，alias 不能。 

七、Ruby 中 Require,Load,Include,Extend 和 Autoload的区别？ 
1、require 方法让你加载一个库，并且只加载一次，如果你多次加载会返回 false。只有当你要加载的库位于一个分离的文件中时才有必要使用require。使用时不需要加扩展名，一般放在文件的最前面：require 'test_library' 
2、load 用来多次加载一个库，你必须指定扩展名：load ‘test_library.rb' 
3、include：当你的库加载之后，你可以在你的类定义中包含一个 module，让 module 的实例方法和变量成为类本身的实例方法和类变量，他们 mix 进来了。include 不会把 module 的实例方法拷贝到类中，只是做了引用，包含 module 的不同类都指向了同一个对象。如果你改变了 module 的定义，即使你的程序还在运行，所有包含 module 的类都会改变行为。 

4、extend 会把 module 的实例方法作为类方法加入类中 

5、autoload 只有使用到需要的常量或类文件才被加载。

3、git merge 和 git rebase的区别

共同点：用于从一个分支获取并且合并到当前分支
但有不同的工作方式
merge特点：自动创建一个新的commit
缺点是显得分支很杂乱
rebase特点：从公共祖先开始，会合并之前的commit历史
优点是得到了更简洁的项目历史，去掉了merge commit
缺点：如果合并出现代码问题不容易定位，因为re-write了history
如果你想要一个干净的，没有merge commit的线性历史树，那么你应该选择git rebase
如果你想保留完整的历史记录，并且想要避免重写commit history的风险，你应该选择使用git merge 

4、是否理解git的工作原理

直接记录快照，而非差异比较
工作区和暂存区
文件的三种状态：已提交、已修改、已暂存
重要的.git目录
  1. 在工作目录中修改某些文件。
  2. 对修改后的文件进行快照，然后保存到暂存区域。
  3. 提交更新，将保存在暂存区域的文件快照永久转储到 Git 目录中。

几个主要的《 Ruby 风格指南》的影响：
  • 每个缩排层级使用两个空格。不要使用制表符。
  • 不要使用 ; 隔开语句与表达式。推论：一行一条语句。 
  • (、[ 之后，]、) 之前，不要添加任何空格。在 { 前后，在 } 之前添加空格。
  • 除个别情况外方法调用应当使用括号包裹参数，尤其是第一个参数以 ( 开头时，比如 f((3 + 2) + 1)；
  • 三元操作符的每个分支只写一个表达式。即不要嵌套三元操作符。嵌套情况请使用 if/else 结构。
  • ...
几个主要的《 Rails 风格指南》的影响：
  • 当需要为一个 RESTful 资源添加动作时（你真的需要吗？），应使用 member 路由和 collection 路由。
  • 当需要定义多个 member/collection 路由时，应使用块结构。
  • 控制器应该保持苗条 ― 它们应该只为视图层提供数据，不应包含任何业务逻辑（所有业务逻辑都应当放在模型里）。
  • 当只需要通过属性查询单个记录时，优先使用 find_by 而不是 where
  • 当需要处理多条记录时，应使用 find_each。
  • ...
