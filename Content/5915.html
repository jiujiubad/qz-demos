  • Rails 代码重构
  • 2017-10-05 09:38


改进的地方有3处：
- 在终端运行了bundle update，升级了部分gem
- 按照issues里面列出的提示，在app/controllers/jobs_controller.rb中，优化case end的代码风格，原先是这样：
    @jobs = case params[:order]
    when 'by_lower_bound'
      Job.published.order("wage_lower_bound DESC")
    when 'by_upper_bound'
      Job.published.order("wage_upper_bound DESC")
    else
      Job.published.recent
    end

改成了：

    @jobs = case params[:order]
                when 'by_lower_bound'
                  Job.published.order("wage_lower_bound DESC")
                when 'by_upper_bound'
                  Job.published.order("wage_upper_bound DESC")
                else
                 Job.published.recent
                end
- 对于app/assets/javascripts/cable.js中的“ this.App || (this.App = {});“， issue显示“Expected an assignment or function call and instead saw an expression.“.小改了一下，将表达式赋值给一个变量：
job_listing = this.App || (this.App = {});

作业四

  • 编程语言 第二集
  • 2017-10-04 21:12
看rails风格指南的时候，很有感触，对比bad, good两种风格的写法，很快就发现老师从一开始就在培养我们良好的编码风格，赞！！说说几个我觉得自己需要更换的编码风格：
- 循环部分，尽量用find_each, 而不是all.each／each
- 在model中，添加has_many 和 has_one的 associations时，不要忘记加上dependent: :destroy
- Time，使用Time.current, not Time.now
- (、[ 之后，]、) 之前，不要添加任何空格。在 { 前后，在 } 之前添加空格
- 把 when 与 case 缩排在同一层级
- 使用 _ 语法改善大数的数值字面量的可读性【不知道竟然还有这种表示方法】
- 不要使用 for
- 利用“if 与 case 是表达式”的这个特性

  • ActionController::UrlGenerationError
  • Url生成出错，no route matches

class Car
  attr_accessor :name, :modal, :year

  def initialize(name, modal, year)
    @name = name
    @modal = modal
    @year = year
  end

  def name
    @name
  end

  def modal
    @modal
  end

  def year
    @year
  end

  def age
    Time.now.year - @year
  end

  def info
    puts "出场年份：#{@year}\n型号：#{@modal} \n名称：#{@name}"
  end

end

# instance
c = Car.new("Audi", "A5", 2016)
puts "车龄: #{c.age}"
c.info
