学员推荐语

我之前在大学学过编程，毕业后也学过一点，但每次学习都会让我觉得编程晦涩难懂，参加全栈营才发现原来是学习的 方法不对。应该要先学习学习的方法，再去学习，仅仅这一点就为我打开了一扇新的大门，如果没有遇到全栈营也许以后也很难有机会明白这一点。更何况，全栈营帮我克服了编程恐惧症！ - xyy601

来全栈营前，我没有任何基础，也不敢相信我可以学会编程，甚至是在这么短的时间内，可是来到全栈营后，我真的做到这一切了。 - 陈慧娟

全栈营的学习不仅仅是编程，在这里会让你学会分享与交流，让你学会学习，让你学会提升认知系统，让你学会项目管理，让你学会做产品并且会推广产品，让你个人能力飞速般的成长......有许多许多的可能。- 李思成

这次的上课方式和以前我接触的很多线上课程都不一样，很多其他课程只是单纯教你技能，而不是教你做人，眼光不够长远，只是单纯迎合市场。这次的学习其实最大的感慨还是分享和帮助——这些软技能的应用。短短两三个月的学习，收获很大，改变很多。感谢xdite老师和其他助教老师！- 隋璐

三个月来，自己真的感觉成长了许多，这不是能用金钱来衡量的。 1.课程设计合理。 2.老师非常牛。 3.效果非常好。 4.学生也非常牛。 5.是一次破茧重生之旅。 - 建清

从来没有体会过用正确的学习方法极速学习的感觉。 体验过之后人生的学习效率极大的提高，相当于间接性地延长了寿命，这种好事怎么能不分享 - 李海豪

观念升级

全栈从字面上讲是一种等级。但是通过这两个月的学习，我觉得全栈是一种人生态度。首先要明白，无论课程是两个月还是两年，课程结束后，需要学习的东西还有很多。你会的工具越多，掌握的技能越多，只会变得更忙碌，很简单啊，因为你能做的事情更多了啊。你能干的事情更多了，你会发现不会做的、不懂的也会更多，你就会发现需要学的东西更多了。 查看原文

从“每天认真工作，正常上下班，也许会慢慢磨练成专家” 变成 “只有上瘾，忘记时间，才可能成为大牛”。 查看原文

相信上过元学习课程的人都已经了解的特别清楚，我在这儿就不多说了。但是呢，我们在全栈营学习的内容是元学习课的一个践行版。学到的再多的理论东西，不通过践行都不是自己的。还有元学习课，只是我们全栈营里学习内容的不到百分之一的内容。 查看原文

追求成长不是目标，这是一种状态，你没有这种状态，凭什么你能获得梦想的东西？不放弃安全感，不对一件自己感兴趣的事情拼命，你怎么可能获得成就，怎么可能获得常人没有的能力？ 查看原文

编程学习

我以为来参加的是培训班，其实是参加了一个大型的自学训练营。一群被应试教育虐大的中国学生，在这种招式下，被激发出巨大的自学潜能。我几乎每天都会被大家一起自学产生的能量所震撼，甚至有一些开学时候基础还比较差的同学，在期中的时候已经变成了小组里的大神。每天写的日记，每周进行的小组总结，都帮助我们加倍地成长，我们的成长速度至少乘了21倍吧（班里一共21个同学）。 查看原文

前面提到，全站营好像乌托邦一样不真实，其实听起来更不真实的是，老师竟然毫无保留地教——业内十几年才总结出的经验，或是花了几十万学到的知识，倾囊相授。这些知识被老师打包成高浓度的“海洛因”，有的，即使是行业大牛也未必了解，而我们作为高级新手，却已经略知壹二，真是叫人拍大腿的痛快。当然，痛快之余，我们也不忘提醒老师给后面上课的同学涨学费…… 查看原文

如果说我曾经怀疑自己成为一个全栈工程师的潜力，在做完这6个项目之后，我不再有任何怀疑。尽管我现在还不足以称自己为“全栈”，但我已经具备了开发最基础功能的 Web + Mobile 前端和后端技能（从这里延伸到iOS/Android也提供了快捷的方法和教材）。更重要的是，我已经拥有了自学开发任何其他简单功能的基础。不喜欢写日记的我，也攒下了近60篇博客，其中约10篇是关于开发某个功能的方法或总结。 查看原文

在这个课程里面其实并不是老师教什么你就学什么，而是你想学什么老师就教什么。毫不夸张。如果你想学到任何东西，就向老师问，老师一定会给你serendipity。不问的那些同学，他会错失几百亿。 查看原文

产品开发

从一个技术的门外汉，到能独立开发极简程序、共同开发完整可上线程序，能够看懂Stackoverflow上的交流贴并自己实现没有做过的新功能、解决没有碰到过的bug，这一切对我来说，其实已经远远超出了我对自己“程序员”身份的假想。 查看原文

在我上一次创业当中，开发一款iOS的App，从立项到内测花了近4个月时间，用了1个iOS、1个后端、1个后端实习生、1个产品经理、1个外包设计师、1个全职设计师，开了几十万的工资。而现在的我，再继续练习1-2个月之后，可以1个人完成那6个人做的所有事情。 查看原文

团队协作

这四周的成长更是飞速的，老师说团队合作就是要跟不同时期的你合作。团队合作会把你的缺点暴露无遗，而你也无法回避它们。你只有去改变才不会拖团队的后腿。 查看原文

在全栈的问题上，我们现在做过的项目却是不多，Code 经验是要通过项目积累的，但是比如团队协作，比如UserStory拆 票，使用 Redmine 项目管理这些东西，很多程序员工作三年都搞不懂这些，而 Code 经验反而是能在工作以后快速提高的。 查看原文

学习方法

所有的编程学习过程其实都有套路可循，既没有想象的那么困难，也没有想象的那么简单。在7月初的时候，我们甚至连 Ruby on rails 是什么都不懂，而到了现在，大家能自己写 CRUD、自己做功能、自己 Debug 、自己做出小型购物网站，甚至开始团队合作开发项目。查看原文

我已经爱上了这种学习模式，如果你让我拿着 Ruby 基础教程这本书一直看，看到现在，我估计还是啥都做不出来，而这种学习模式让我现在有 10000% 的信心自己可以学好 Rails 甚至更多的东西。 查看原文

那我作为一个初学者，我就有了两种方法来学 Rails 啊。一种是像人类研究自然一样，观察，观察，连接输入和输出的规律，有了积累之后再去找个理论揣测黑箱的模式；或者，直接了解黑箱，了解完了黑箱，自然能够判断输入和输出的规律。现在 xdite 在带着我们做的事情呢，就是对一个复杂的人工系统做类似于科学观察的黑箱探索。查看原文

课程进行到现在，最大的收获应该是学习方法上的，因为这是可以举一反三的东西。用一个字来概括就是“用”，在知道了最少必要知识之后就去实战，然后在实战中再不断补充需要的知识，狠有成效。查看原文

产品与心态

Rocket 项目从开始到如今近3周了，在没有前端，没有设计的 5 个 RoR 新手努力下，居然进展了 80%，到了修 Bug、完善用户体验的阶段，实在是很神奇。查看原文

追求做出完美的东西其实是一件不可能的事情。你觉得完美的东西其实可能就是高你 N 个 Level 的作者眼中不完美的产品。所以，重点不是做出现阶段你觉得完美的东西（当然这本身也是不可能的），而是去向更好的层次攀登，品味高了，质量自然就高了。查看原文

跳出困局的方法是尝试比你优秀的人告诉你的高阶方法。有时候可能受困于自己的经验，但你可以尽量去尝试，做过之后你就反过来明白为什么大神让你那么做，你可能就跳出困局了。查看原文

不要因为急于得到大家的认可，去做那些很炫的东西浪费时间，要专注于 Basics ，也就是最重要的那些东西；决策的标准只有一个，就是做这件事对项目有利。查看原文

团队合作

由于每个人负责的模块不同，虽然我不能亲自实现某个模块的某个功能，但代码在团队内是公开的，我们可以互相学习其他队员写的好的地方， xdite 老师的代码批改也会让我们学习到某些编码方式的最佳实践。所以团队合作也是一种“学习方式”。查看原文

我发现在一个良好的协作氛围中，可以更快速地学到很多东西。开会讨论问题的时候，队友们的思路都很广。以前只是作为用户去用一些产品，现在要从开发者的角度来思考问题，所以要多多向他们学习。查看原文
